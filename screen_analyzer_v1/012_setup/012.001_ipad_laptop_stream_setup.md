# 012.001 iPad-to-Laptop Camera Stream Setup (Windows 11 + iPad Pro)

This guide walks through setting up an iPad camera stream so `screen_analyzer_v1` can read it as a camera source on a Windows 11 laptop.

## Goal
Use the iPad as an external camera and make the stream readable by OpenCV in this project.

Recommended V1 path:
- iPad app publishes RTSP stream
- Windows laptop runs local RTSP server relay
- app reads `rtsp://127.0.0.1:8554/ipad`

---

## Prerequisites

- Windows 11 laptop on the same Wi-Fi network as the iPad
- iPad Pro with camera permissions available
- Existing Python venv setup completed (`011_env`)
- `screen_analyzer_v1` dependencies installed

Optional but strongly recommended:
- Keep laptop and iPad plugged in during long runs
- Disable iPad auto-lock while streaming

---

## Step 1) Install and run MediaMTX on the laptop

MediaMTX is a lightweight RTSP server that receives the iPad stream.

1. Download the Windows build of MediaMTX.
2. Extract it to a folder such as:
   - `C:\tools\mediamtx\`
3. Open PowerShell in that folder and run:

```powershell
.\mediamtx.exe
```

4. Leave this terminal open while streaming.

Default RTSP listener is `:8554`.

---

## Step 2) Find laptop local IP address

In PowerShell:

```powershell
ipconfig
```

Use the active Wi-Fi IPv4 address (example: `192.168.1.42`).

---

## Step 3) Configure iPad streaming app (Larix Broadcaster)

Install **Larix Broadcaster** on iPad.

In Larix, create a new connection:

- URL: `rtsp://<LAPTOP_IP>:8554/ipad`
- Protocol: RTSP
- Video codec: H.264
- Resolution: start at `1280x720`
- FPS: `15` (or `30` if network is stable)
- Audio: off (not needed for this project)

Start broadcasting from iPad.

---

## Step 4) Allow firewall traffic on Windows (if needed)

If stream fails to connect:

- Allow `mediamtx.exe` through Windows Defender Firewall
- Or open inbound TCP port `8554` on Private network profile

Then retry broadcast from iPad.

---

## Step 5) Verify stream with smoke capture script

From `screen_analyzer_v1`:

```powershell
python .\005_scripts\005.002_smoke_capture.py --source "rtsp://127.0.0.1:8554/ipad" --output-dir .\008_outputs\smoke
```

Expected behavior:
- Preview window opens
- Press `s` to save a frame
- Press `q` to quit

If preview opens and a frame saves, camera ingestion is working.

---

## Step 6) Set runtime config for full app

Edit `002_config/002.002_runtime_settings.yaml`:

```yaml
camera:
  source: "rtsp://127.0.0.1:8554/ipad"
  source_type: "auto"
  startup_timeout_seconds: 30
```

Keep V1 defaults unchanged unless needed:
- `capture.interval_seconds: 3`
- `capture.max_in_flight_requests: 1`
- `capture.latest_frame_wins: true`

---

## Step 7) Run the desktop app

```powershell
python .\001_app\002_app_entrypoint.py
```

The dashboard should show:
- live feed
- status updates
- latest analysis result
- errors (if any)

---

## Troubleshooting checklist

### No video / connection timeout
- Confirm MediaMTX is running
- Confirm iPad and laptop are on same network
- Confirm iPad stream URL uses laptop IP, not `127.0.0.1`
- Confirm Windows firewall rule allows RTSP traffic

### Works in smoke test but not full app
- Confirm `camera.source` in `002.002_runtime_settings.yaml`
- Confirm app is started from `screen_analyzer_v1` root
- Check `010_logs/runtime.jsonl` for camera and analysis errors

### Stream is choppy or drops
- Lower iPad resolution to `960x540` or keep `1280x720`
- Use `15` FPS instead of `30`
- Move devices closer to router or use less congested Wi-Fi

### iPad stops streaming after a while
- Disable auto-lock
- Keep app in foreground
- Connect power during long runs

---

## Alternate direct source options (optional)

The app can also read:
- `http://...` MJPEG URLs
- direct device index (`0`) for USB webcams

For this project and Windows 11, RTSP with MediaMTX is usually the most stable path.
